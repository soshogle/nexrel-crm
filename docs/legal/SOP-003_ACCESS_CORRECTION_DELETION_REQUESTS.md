# SOP-003 - Access, Correction, and Deletion Requests

Document ID: SOP-003  
Owner: Privacy Officer  
Effective Date: ****\_\_****  
Review Cycle: Annual

## 1. Purpose

Provide a compliant process for handling data subject rights requests (access, correction, deletion where applicable).

## 2. Intake Channels

- Email: privacy@soshogle.com
- In-app/support ticket intake
- Written request channels approved by operations

## 3. Procedure

1. Log request ID, date/time, requester details, and request type.
2. Verify identity before data disclosure.
3. Acknowledge request promptly.
4. Collect relevant records and disclosure history.
5. Respond within 30 days; if extension required, notify within original 30-day period with reason and recourse notice.
6. Provide data in understandable format and explain abbreviations/codes.
7. Process correction/deletion decisions and document outcomes.
8. Notify third parties of corrections where appropriate and lawful.

## 4. Refusal/Partial Refusal

Any refusal must be in writing with:

- legal basis/reason,
- available recourse,
- complaint option details.

## 5. Records to Retain

- Request log
- Identity verification evidence
- Response package metadata
- Decision rationale
- Third-party notifications (if sent)
